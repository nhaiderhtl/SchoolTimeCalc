# Plan: 03-01-PLAN.md - Holiday Data Models & API Client Expansion

## Goal
Extend the data layer and API client to support fetching and storing holiday data directly from the WebUntis API.

## Requirements
- SYNC-03

## Steps
1. **Data Models:**
   - Create a `Holiday` entity class in the core domain (e.g., `Id`, `Name`, `StartDate`, `EndDate`, `SchoolId`).
   - Add a `DbSet<Holiday>` to the application's `DbContext`.
   - Configure Entity Framework Core relationships and constraints (e.g., unique indexes on StartDate/EndDate/SchoolId).
   - Generate and apply a new database migration (`dotnet ef migrations add AddHolidays`).
2. **API Client Expansion:**
   - Update the existing Refit interface (e.g., `IWebUntisApi`) to include the JSON-RPC method for retrieving holidays (typically `getHolidays`).
   - Define the corresponding Request/Response DTOs mapping to the WebUntis JSON structure for holidays.
3. **Unit Tests:**
   - Add basic unit tests to verify the JSON serialization/deserialization of the new holiday DTOs.
