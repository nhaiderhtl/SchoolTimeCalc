# Plan: 03-03-PLAN.md - Integration & Settings UI Updates

## Goal
Integrate the new holiday synchronization capabilities into the application's user interface, allowing users to view sync status and manually trigger updates.

## Requirements
- SYNC-03 (UI representation)

## Steps
1. **Sync Status UI:**
   - Update the Blazor settings page (e.g., `Settings.razor` or similar) to display the "Last Synced Holidays" timestamp.
   - Show a count of successfully synced holidays (e.g., "30 holidays cached").
2. **Manual Trigger:**
   - Add a "Sync Holidays Now" button to the settings UI.
   - Wire the button to a backend endpoint or Blazor server method that invokes the `IHolidaySyncService.SyncHolidaysAsync()` method.
3. **Error Handling & Notifications:**
   - Implement loading states (e.g., a spinner while syncing).
   - Display a success message or an error alert if the WebUntis API rate-limits the request or fails authentication.
4. **UI Validation:**
   - Write bUnit component tests to verify the UI displays the correct sync status and the button click triggers the expected service call.
