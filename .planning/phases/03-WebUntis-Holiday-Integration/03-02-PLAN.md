# Plan: 03-02-PLAN.md - Holiday Sync Service

## Goal
Implement a reliable background service or synchronous process to pull holiday data from WebUntis and persist it securely to the application's PostgreSQL database.

## Requirements
- SYNC-03

## Steps
1. **Sync Service Interface:**
   - Define an `IHolidaySyncService` interface to handle fetching and saving.
2. **Service Implementation:**
   - Create a `WebUntisHolidaySyncService` that orchestrates:
     - Fetching the current authenticated user's school holiday calendar from WebUntis using the updated API client.
     - Mapping API DTOs to the `Holiday` database entity.
     - Performing an "Upsert" operation: saving new holidays, updating changed ones, and removing any no longer present in the API.
3. **Background Jobs / Triggers:**
   - Set up Hangfire or an IHostedService background job to run the holiday synchronization periodically (e.g., once daily) for active users.
   - Alternatively, tie it to the same background task that updates the base timetable.
4. **Integration Testing:**
   - Write integration tests using `WebApplicationFactory` and a mock WebUntis endpoint to ensure the sync process correctly writes to an in-memory or test database.
